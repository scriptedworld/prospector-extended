# Task 4.07: Extend complexipy_tool to emit metrics

## Metadata

- **Phase:** 4 — Metrics Integration
- **Task ID:** 4.07
- **Type:** tdd
- **Priority:** medium
- **Depends on:** 4.01

## Description

Extend the existing complexipy tool wrapper to emit per-function cognitive complexity scores for ALL functions (not just violations above threshold) into the metrics block.

## Acceptance Criteria

- [ ] All per-function cognitive complexity scores emitted to `metrics.complexity.cognitive`
- [ ] Includes: file, function name, class (if method), line, score
- [ ] Aggregate stats: max, mean, functions_above_threshold
- [ ] Existing violation behavior unchanged (still emits Messages for functions above threshold)
- [ ] Backwards compatible — no change to `messages` output

## Current Behavior

Currently `complexipy_tool.py` iterates function results and only creates `Message` objects for functions exceeding the threshold. Scores for compliant functions are discarded.

## Change Needed

After the violation check loop, collect ALL function scores into a metrics list and register with the `MetricsCollector`.
