# Task 4.01: Metrics Output Infrastructure

## Metadata

- **Phase:** 4 — Metrics Integration
- **Task ID:** 4.01
- **Type:** tdd
- **Priority:** high
- **Depends on:** None (foundation task)

## Description

Add a `metrics` block to the prospector-extended JSON output alongside the existing `summary` and `messages` blocks. This is the foundation that all subsequent metrics tasks build on.

## Acceptance Criteria

- [ ] `MetricsCollector` class that tools can register metrics into
- [ ] `metrics` key appears in JSON output when `--output-format json` is used
- [ ] Metrics block has top-level sections: `project`, `complexity`, `maintainability`, `structure`, `coverage`, `dead_code`, `duplication`, `suppressions`
- [ ] Empty/default values when no tools have contributed metrics yet
- [ ] Existing `summary` and `messages` output unchanged (backwards compatible)
- [ ] CLI `--metrics` flag to enable/disable metrics collection (default: enabled)

## Design Reference

See `docs/design/metrics-integration.md` for the proposed JSON structure.

## Notes

This task creates the plumbing. Subsequent tasks (4.03-4.15) wire individual tools into it.
