# Task 4.05: Add StructureTool (AST-based metrics)

## Metadata

- **Phase:** 4 — Metrics Integration
- **Task ID:** 4.05
- **Type:** tdd
- **Priority:** high
- **Depends on:** 4.01

## Description

Add an AST-based analysis tool that extracts structural metrics not covered by radon: function length, nesting depth, parameter counts, and public/private visibility classification.

## Acceptance Criteria

- [ ] `StructureTool` registered in tool registry
- [ ] Per-function metrics: line count, parameter count, max nesting depth
- [ ] Per-class metrics: method count, line count
- [ ] Visibility classification: public vs private (leading underscore) for functions, classes, methods
- [ ] Aggregate stats: max/mean function length, functions above threshold, max nesting, max params
- [ ] Populates `metrics.structure` block
- [ ] No violations emitted (metrics-only) — returns empty `messages` list
- [ ] No external dependencies (stdlib `ast` module only)

## Metrics Produced

```json
{
  "functions": {
    "total": 80,
    "public": 50,
    "private": 30,
    "max_length": 50,
    "mean_length": 15,
    "above_60_lines": 0,
    "max_params": 3,
    "above_3_params": 0,
    "max_nesting": 3,
    "above_nesting_threshold": 0
  },
  "classes": {
    "total": 17,
    "public": 17,
    "private": 0
  }
}
```

## Implementation Notes

- Walk AST with `ast.NodeVisitor`
- Function length = `node.end_lineno - node.lineno + 1`
- Nesting depth = track depth of nested `if/for/while/with/try` inside functions
- Visibility = `name.startswith("_")` for private
- Consider `@staticmethod` / `@classmethod` as methods, not standalone functions
