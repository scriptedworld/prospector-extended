# Task 4.09: Extend interrogate_tool to emit metrics

## Metadata

- **Phase:** 4 — Metrics Integration
- **Task ID:** 4.09
- **Type:** tdd
- **Priority:** medium
- **Depends on:** 4.01

## Description

Extend the existing interrogate tool wrapper to emit docstring coverage metrics (percentage, per-type breakdown) into the metrics block, not just violation messages for missing docstrings.

## Acceptance Criteria

- [ ] Overall coverage percentage emitted to `metrics.coverage.docstrings.percentage`
- [ ] Total nodes and documented count emitted
- [ ] Per-type breakdown: module, class, function — each with total and documented counts
- [ ] Existing violation behavior unchanged (still emits Messages for missing docstrings)
- [ ] Backwards compatible

## Current Behavior

Currently `interrogate_tool.py` only creates `Message` objects for missing docstrings (INT1xx codes). Coverage percentage and per-type counts are not surfaced.

## Change Needed

After collecting violations, also compute and register:
- Total documented / total nodes = percentage
- Breakdown by node type (module, class, function, method)
