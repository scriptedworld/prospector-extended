# Task 4.03: Add RadonTool (cc + raw + mi)

## Metadata

- **Phase:** 4 — Metrics Integration
- **Task ID:** 4.03
- **Type:** tdd
- **Priority:** high
- **Depends on:** 4.01

## Description

Add radon as a prospector-extended tool wrapping three subcommands: cyclomatic complexity (cc), raw LOC metrics (raw), and maintainability index (mi). Radon is already available as a transitive dependency.

## Acceptance Criteria

- [ ] `RadonTool` registered in tool registry
- [ ] `radon cc` → per-function cyclomatic complexity with rank (A-F), populates `metrics.complexity.cyclomatic`
- [ ] `radon raw` → per-file LOC/SLOC/comments/blanks, populates `metrics.project`
- [ ] `radon mi` → per-file maintainability index with rank, populates `metrics.maintainability`
- [ ] No violations emitted (metrics-only tool) — returns empty `messages` list
- [ ] Configurable via `.prospector.yaml` (enable/disable individual subcommands)
- [ ] Graceful handling if radon not installed (skip with warning)

## Implementation Notes

- claude-scripts already has radon wrappers in `core/metrics.py` — reference for output format
- Use radon's Python API directly rather than subprocess (if available), fall back to subprocess
- radon cc `--json` output: `{"file.py": [{"type": "method", "rank": "B", "lineno": 40, "complexity": 7}]}`
- radon raw `--json` output: `{"file.py": {"loc": 80, "lloc": 41, "sloc": 45, "comments": 6, "blank": 17}}`
- radon mi `--json` output: `{"file.py": {"mi": 79.96, "rank": "A"}}`
