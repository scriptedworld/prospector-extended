## Description

End-to-end integration tests verifying the complete metrics output when running prospector-extended against a test fixture project.

## Acceptance Criteria

- [ ] Test fixture: small Python project with known metrics (specific function lengths, complexity, etc.)
- [ ] Full JSON output includes `metrics` block with all sections populated
- [ ] Metrics values match expected values for the fixture project
- [ ] `--no-metrics` flag produces output without `metrics` block
- [ ] Backwards compatibility: existing `summary` and `messages` unchanged
- [ ] Metrics are deterministic (same input = same output, no timing/ordering variance)
- [ ] Each metrics section tested independently (radon, structure, complexipy, interrogate, vulture, suppressions, duplication)
